> Just one more comment to the REST access: If the following is the

>     http://mrs.cmbi.ru.nl/mrsws/search/rest/GetEntry/db/sprot/id/104k_thepa/format/plain
>
>
> should perhaps also the parsers (in their 'pp' methods) use this way
> when creating the hyperlinks to entries in other databases?

No, this is a link intended to be used by software, not by humans.

> I understand
> that the result, as it is now, would need to be slightly changed because
> it is wrapped in the SOAP's body tags now. But such links would follow
> the "official" way and would not rely on the actions (such as
> "entry.do") from the mrsweb application.

The REST and SOAP interfaces are the API to use by other software, the 
web-app might need some 'mounted' URL's to display single entries as 
well. If it is considered really useful I can create something like that.

But what you're asking for is something else and I really need to add 
this functionality to the MRS web application. I think the best would be 
to create a pseudo page that knows where to link. Then you can create 
links in the parser to this 'page' and the web app will redirect you to 
the right location.

Another option might be to agree on some kind of mark-up in the returned 
HTML that allows the web-app to recognise databank links and create 
correct URL's for them. Something like this:

<mrs:link db="sprot" id="104k_thepa" acc="P15711" 
xmlns:mrs="http://mrs.cmbi.ru.nl/db-link"/>

Then the web-app can decide what to do with this.
