Martin,

You're close.

Parser scripts are used during indexing to split the input data stream 
into 'documents'. These documents are stored in the .cmp file MRS 
creates. During this indexing phase, a parser script also feeds MRS the 
data it needs to index, this is done using the IndexValue and IndexText 
calls in the parser script.

This means, you can throw away the raw data if you want to after 
indexing. However, the other scripts in the update system depend on the 
raw files being there to find out if there is anything to do.

The web service application mrs-ws uses the parser scripts to generate 
HTML (and sometimes to create FastA output). For XML I've decided to use 
XSLT to simplify this process. I used to use XML::XSLT for this, but 
since it is seriously restricted, I've decided to replace it with 
XML::LibXSLT.
