And I reserve the right to change it as often as I like, I hope you will 
understand. That's why I've implemented both a SOAP and a REST 
interface, those are guaranteed to remain the same in the future, the 
web application will change.

E.g., to fetch an entry you should use SOAP, or if that's too much 
trouble you can use the REST style call:

http://mrs.cmbi.ru.nl/mrsws/search/rest/GetEntry/db/sprot/id/104k_thepa/format/plain

If you however use:

http://mrs.cmbi.ru.nl/mrs-web/plain.do?db=sprot&id=104k_thepa

Then you're using an undocumented feature that is certain to stop working.
