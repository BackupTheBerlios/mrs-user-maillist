Well, in a way it is documented. The REST style interface is based on 
the SOAP interface as it is generated on the fly from the same code. I 
may even create a WSDL 2.0 for it, once I have the time to find out how 
I'm supposed to do that.

But it really is very simple, take a look at the WSDL and there you find 
the names of all functions and their parameters. For all the 'simple' 
calls (calls that only take basic input parameters, no structures) you 
can construct the REST URL as was given the example:

First the machine address and the (reverse proxied port):

http://mrs.cmbi.ru.nl/mrsws

(might also be e.g. http://localhost:18081/)

Then the path to the rest call:

rest/GetEntry

And then the parameters as name/value pairs:

db/sprot and id/104k_thepa

Together:

http://mrs.cmbi.ru.nl/mrsws/rest/GetEntry/db/sprot/id/104k_thepa
