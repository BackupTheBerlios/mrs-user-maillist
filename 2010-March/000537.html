I suddenly realised I completely forgot to answer this question.

Op 15-03-10 09:54, Martin Senger schreef:
> I am writing now some parser scripts (e.g. for MicroRNA databases) and 
> I wonder what is the recommended way for creating the HTML hyperlinks 
> (in the parser's "pp" method). I would like that the parser recognizes 
> whether a database (where a link should point to) is available in the 
> current MRS server or not. In the latter case, the parser can create a 
> link to an external (native) database, in the former case the parser 
> will point to the same MRS server (and gets much faster response when 
> a user click on it in mrsweb). What would be the best way to test such 
> condition in the parser?

This is currently a bit difficult to do. What I need to do is extend the 
interface of MRS as it is exposed in the script files. That way you can 
query MRS for information like whether a databank is indexed or a record 
actually exists. Unfortunately, this is not currently possible. It's not 
that much work, but I haven't had time to do it yet.

For now one option might be to use the xml config file to find out if a 
databank is enabled in your setup. You could load that file using 
XML::Simple and then query the information in it. Do this in the 'new' 
routine of the script and keep the information cached in the parser object.
