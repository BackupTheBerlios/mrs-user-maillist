You're almost there.

At the CMBI we've boost 1.42 compiled from source code. To make sure I 
don't mess up, I removed all boost code installed by apt.

The boost was built in my home directory and installed in 
$HOME/projects/boost

I also used the bjam option to only build static libraries. That way I 
can share the resulting executables with others.

I believe the command to do this was:

bjam --prefix=$HOME/projects/boost link=static threading=multi \
	stage install

After this, you should rebuild the libzeep, to make sure it uses the 
same boost libraries.

And then you should build MRS. You might try to get the configure script 
running, but if you have things laid out as I have, you can also use 
this make.config file I have. (configure.pl creates a make.config file, 
and reads it when you rerun configure.pl).

I hope this helps.

-maarten
