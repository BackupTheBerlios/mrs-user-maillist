Please check out https://mrs.cmbi.ru.nl/mrs-3-test/

That's a new versions supporting the new query syntax.

To summarise:

- plain words are used as ranked query terms
- all other constructs are interpreted as boolean filters
- a simple boolean filter is 'indexname:value' which obviously looks  
for value in indexname.
- whitespace is implicitly interpreted as an AND operator
- '|' is the OR operator
- groups can be formed using parenthesis '(' and ')'
- everthing inside a group is interpreted as a boolean query filter,  
if no indexname is specified all indices are used.
- Not is specified by a leading minus ('-') character.

One detail remaining is that the current implementation allows for  
whitespace between the NOT operator and the following terms/ 
construct. I personally don't like that, but it is so ugly to  
implement code that works around this (doesn't make sense, I know,  
the real reason is of course that I'm lazy).
