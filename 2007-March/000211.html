On Fri, Mar 02, 2007 at 08:53:54AM +0100, Maarten L. Hekkelman wrote:
> When we choose to accept AND, OR and NOT at least these will have to  
> be case sensitive. Otherwise the grammar gets too ambiguous.
> 
> But is it an option to allow only '-' and '|' as operators (along  
> with parenthesis of course). Your query would then be entered like:
> 
> "os:carica os:papaya (de:protease | de:proteinase)"
> 
> Is that acceptable?

Well, IMHO it does not really matter whether you use ! or - or NOT for 
the logical "but not", as long as it works. I however know by experience 
that end users hate changes in the way their tool works, certainly if it 
is just for doing the same thing in a different way. So, it would be nice 
if the MRS query language could become "stable" (except for extensions) 
as fast as possible. And, of course, the language should be the same at 
all levels (Web interface, Web services, direct access with Perl script).

By the way, what about the feature in MRS2 where you could e.g. search 
for "abnormal auricles" ?
