> Another problem is with enzyme. In an entry like e.g. 3.4.22.2  
> there is a
> line
> DR   P00784, PAPA1_CARPA;
> and the link entry.do?db=uniprot&query=ac:P00784 does not work.
> I am however amazed and do not understand what is wrong.

The link generated should read:

entry.do?db=uniprot&id=PAPA1_CARPA

or

query.do?db=uniprot&query=ac:P00784

The first link retrieves one entry but that fails if there is no  
entry ID specified (or the ID does no longer exist).

The second link does a real query for one accession code MRS 3.0 then  
display the found entry if it is the only one. (BTW, I've changed  
this behaviour. In MRS 4.0 if there's only one hit the entry will be  
displayed only when the search term was the ID for this entry).
