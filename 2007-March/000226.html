Op 16-mrt-2007, om 14:26 heeft Alex Patak het volgende geschreven:

> Thanks, this helped me to get the .cmp file.
>
> During the execution of the script I got several errors like "Use  
> of uninitialized value in..."  (Find attached the hole list of  
> errors).

Those are warnings coming from the genbank.pm perl script. I've set  
the script to some kind of paranoia mode to detect errors early.

To avoid many of them you can assign the environmental variables  
MRS_DATA_DIR, MRS_RAW_DIR and MRS_SCRIPT_DIR in your shell before  
running mrs.pl.

> I have following extra questions:
>
> 1) During this process no .dict file was created like the update  
> scripts do. What is this .dict file?

dict files can be created using the 'mrs.pl dict' command.

If you run mrs.pl without any argument it will print out most of the  
options available.

A .dict file is used by the web application to generate a list of  
spelling alternatives for query terms.

> 2) Would the same procedure work with EMBL formatted sequences?

In theory, yes.

> 3) I added the database name to the /usr/local/etc/mrs-config.xml  
> file but I get an empty line in the popup menu of the web  
> interface. Is there somewhere else where I have to declare the new  
> database?

Ah... yes. well... To add a name to your databank you have to edit  
the genbank.pm script. At line 66 in this script you see a hash that  
maps the databank short name to a human readable long name. Just add  
your own name here and recreate your databank with the same command.
