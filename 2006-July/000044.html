Op 17-jul-2006, om 11:27 heeft Laurent Falquet het volgende geschreven:
>
> After spending several hours installing MRS, I
> finally managed to compile it the following way:

I'm sorry it took you so long.

> - install a fresh Fedora core4 on an old PC.

An old PC is not the right PC to use with MRS. Hosting MRS is not a  
problem, but indexing really requires a lot of CPU horsepower.

> - run install.pl and install the HUGE!!! list of all missing modules

You do get a virgin perl with core 4 then...

> - try to make install in the "plugin" directory,
> that was the most difficult and annoying part.
>     my fresh Fedora was missing the correct SWIG,
> BOOST and GCC, but unfortunately these
>     were not seen as outdated by YUM. Thus I had to update them  
> manually...

SWIG is not needed for a simple recompile. There may be a warning  
about a missing swig but that warning should contain a statement that  
this is not fatal.

> - after updating these tools, the compilation went OK.
> - modify httpd.conf and restart apache. This was OK.
> - try to "make" a database available...
>     I tried both UniProt and Prosite, but
> unfortunately it didn't work, and I am stuck here:
>
> more /usr/local/data/status/prosite.make_log
> make[3]: Entering directory `/usr/local/share/mrs/update_scripts'
> make[3]: Nothing to be done for `data'.
> /usr/bin/perl /usr/local/bin/mrs.pl create -d prosite
> Can't locate object method "version" via package
> "prosite::parser" at /usr/local/bin/mrs.pl line
> 290.
> Parsing done, creating index... make[3]: ***
> [/usr/local/data/mrs/prosite.cmp] Illegal
> instruction

Apparently MRS is crashing on your computer. Since you were talking  
about an old computer, what CPU does this computer have? Illegal  
instruction often means it is compiled for another CPU than the one  
you're using. It might very well be that you e.g. have a PIII and try  
to run a P4 compiled application on it. That won't work.

And then, sometime ago someone at GNU headquarters decided it was a  
good idea to rename gzcat to zcat. You can try to edit the GZCAT  
variable in your make_xxx.conf file to point to your copy of zcat.
