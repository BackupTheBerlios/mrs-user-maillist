>
> So the questions are:
> 1) How does MRS find out which databases exist, to list them on the  
> overview page?

It first looks at the mrs.conf file and then tries to open each file  
listed therein. It only displays the files that succeeded.

> 2) What can be wrong with the mrs files?

Either they are OK, or indexing failed. Do you have any error  
messages in your log files?

You can try to use the mrs.pl script to find out what's in a .cmp  
file by using something like this:

$ perl /usr/local/bin/mrs.pl info /data/mrs/sprot.cmp

This will give you a dump of the information stored in the cmp file.  
If this fails, the file is really corrupt and should be deleted.
