Uniprot is not a databank in itself, it consists of two separate MRS  
files: sprot.cmp and trembl.cmp. To query uniprot you have to specify  
'sprot+trembl' to the MRS plugin.

In mrs.conf (in the cgi-bin directory) the entry for Uniprot should  
contain an "id => 'sprot+trembl'" for the UniProt entry. In earlier  
releases this erroneously contained "id => 'uniprot'" since the  
concatenation operator + did not work at the time the mrs.conf file  
was first created.

I hope this information is what you've been looking for. If there's  
something else going wrong, please let me know.
