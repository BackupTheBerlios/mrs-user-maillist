Hi,
I finally found out how to get the BLAST jobs started on the Debian 
(Ubuntu) box. On this machine, the batch command does not take 
arguments, while the mrs-submit.cgi says in essence "batch -f queryxxx.csh".
1. I had to put www-data in the at.allow file, as this is the user name 
for the apache job.
2. I replaced the "-f" parameter with "<" in line 165 of the 
mrs-submit.cgi file
In this way, the BLAST jobs start readily on the Debian configuration. 
On SuSE and Gentoo, the line is valid as is.
The at.allow and at.deny files have to be adapted in all cases, as you 
may guess.
