Op 11-jul-2006, om 17:20 heeft Dr. Andrea Schafferhans het volgende  
geschreven:
>
> after installing boost with the standard parameters (bjam "- 
> sTOOS=gcc" install), the boost installation is still not found. The  
> message I see is "You do not seem to have the TR1 headers  
> installed". If I look in the install directory (/usr/local/include/ 
> boost...), I can find the file version.hpp there. Likewise, the .a  
> and .so files of boost can be found in the directory that is  
> specified in the $LD_LIBRARY_PATH (/usr/local/lib).
> So do you have any tips what parameters I have to set such that  
> boost can be found?

TR1 and boost are not the same. TR1 is part of gcc version 4.1 and  
higher. That's not really a problem, if boost works you can build the  
plugin as long as gcc is at least version 3.4.

A version 4.1 gcc is preferred though.
