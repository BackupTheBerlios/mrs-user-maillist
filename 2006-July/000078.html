In the mrs-submit.cgi script you can find some lines that try to  
start a job in a queueing mechanism. Some of these are commented out,  
the one currently active uses the 'batch' command of your Linux  
system. But for batch to work, you have to enable 'at'. There was a  
posting on this list from Robert recently describing what he needed  
to do in order to get at to work.

To have a look at the current at queue there is a command called  
'atq' and usually at runs every 5 minutes. That means your job might  
wait until the next at run and that can take up to 5 minutes. You  
have to config at to run in smaller intervals.

> To find out more, I also tried to run the query_xxxxx.csh script  
> and its
> individual commands to find out what happens. And what I got was a  
> core:

The core you get is because mrs_blast tried to open your sprot.cmp  
file but wasn't allowed to. Perhaps the permissions on sprot.cmp are  
not correct? You might also make sure that mrs knows where to find  
the mrs files. For this MRS uses the environmental variables  
MRS_DATA_DIR which should contain the path to the MRS files.
