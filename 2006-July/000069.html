Op 26-jul-2006, om 12:50 heeft Dr. Andrea Schafferhans het volgende  
geschreven:
>
> after looking at the output of make -n in the update_scripts  
> directory I
> have the suspicion that my setting of the DATABASES variable in my
> make_xxx.conf file is not used.
>
> Could it be that the line defining DBALLFLAGS in the Makefile should
> refer to the variable $DATABASES instead of referring to  
> $ALL_DATABASES?

No, that should make no difference. The ALL_DATABASES is used only to  
create all the rules for all possible dbs. The db's actually made  
depends on DATABASES.

You can play around with the rule test:

	make test

It should echo the current value of DATABASES, and replace the  
statements in line 115 of Makefile to print out anything you want the  
value of.

Simply running make -n without an argument should run a make all, but  
I'm not sure if that will always work. What argument did you specify?
