The 4.1 release of MRS does not check the input parameters for Blast 
searches. As a result, it may crash when people try to access your mrs 
server using invalid SOAP requests containing e.g. invalid matrix names 
or blastn as program. I've checked in a fix for this in the repository 
and this will show up in a future release. You've been warned.
