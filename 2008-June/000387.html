Hi,

I get the feeling EMBL has grown too large to put into one file. I  
propose to split EMBL into sections. That's easy, based on file name.

The sections would be:

MRSLIBS	= embl_ann_release embl_con_release \
           embl_est_release embl_gss_release \
           embl_htc_release embl_htg_release \
           embl_pat_release embl_std_release \
           embl_sts_release embl_tpa_release \
           embl_wgs_release

I'm also reconsidering the whole merging step. I've created a multi  
threaded indexing application that replaces the current mrs.pl  
script. It makes much better use of the resources of modern computers.

When we split EMBL, there's no need to have a merge step at all anymore.

I hope to have a beta release of the new MRS within three weeks. It  
has taken a long time, but I've been waiting almost a year for the  
new hardware needed to continue development.
