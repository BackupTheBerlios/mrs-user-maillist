> de:cytochrome & de:b5 & oc:viridiplantae ! de:reductase ! de:fragment

This is fixed now. But please note that I had to change the behaviour  
of one of the Perl interface calls to fix this problem.

The MQueryResults::Count call takes a boolean argument. When this is  
true, the iterator is not longer useable to fetch results. That means  
that if you have code like this:

my $db = new MRS::MDatabank('sprot');
my $r = $db->Find('blablabla');

my $count = $r->Count(1);

while (my $hit = $r->Next) {
	...
}

This will no longer work. You have to call Count after fetching your  
results using Next.

At the same time I fixed the query parsing in the web interface. You  
can now add glob style patterns to a query and so

rds -respir*    ( <= results in all records containing RDS but  
excluding those that contain words starting with respir)

works again.
