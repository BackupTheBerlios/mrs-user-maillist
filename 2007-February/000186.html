Hi Maarten,
Sorry, I was again stuck with a Java error when trying to access the web 
interface for a new MRS server on an Ubuntu machine (Debian). I searched 
for some time till I remebered that the security approach for Tomcat 5.5 
on Debian is to enable security (see startup script where the option 
TOMCAT5_SECURITY=yes is given. Changing this to no allows entering the 
web interface.
Be warned that this behaviour might also amper web visibility on other 
distributions.

Now that this is cleared, I got another error, in the top page of the 
MRS web interface:
"Cannot connect to SOAP server at http://localhost:8081/mrs/soap"

I solved this by editing the mrs-config.xml file :
The original string about data location is /data//mrs. I changed it to 
/data/mrs.

This is probably a simple detail for Maarten to fix.

Robert
