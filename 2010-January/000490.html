Martin,

>     The obvious solution of installing an earlier version of gcc does
>     not seem to work unfortunately, I'm afraid there is a binary
>     incompatibility between boost libraries built with gcc-4.4 linked
>     to an application built with gcc-4.3. You still might want to try it.
>
>
> I am using the newest boost library 1.40.0. But Karmic originally came 
> with 1.38.0. Perhaps I should try with the 1.38.0?

If you really want to try, you can apt-get install gcc-4.3 (or maybe 
even better gcc-4.2) and then use this compiler in the configure.pl 
script (-cc=gcc-42 option) followed by 'make clean all'. Then test the 
software, if it crashes you'll have to build boost yourself using the 
same gcc and make sure you *entirely* remove the current boost version 
from disk. I ran into very mysterious problems that I tracked down to 
mismatches between boost header and lib versions and perhaps also boost 
libs built with another compiler than the MRS applications.

But like I said, if you have the time you might want to try to see how 
far you get. Otherwise you'll have to wait until I've released an update.
