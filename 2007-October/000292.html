Do you have more detailed output from this log file? You could also 
check each part separately by running 'mrs.pl info -d 
embl_release-x.cmp' on each to see if there's any output at all or an 
error message. If one is corrupt you might want to delete that part and 
restart the make.

There are a few problems with the current MRS when indexing a large 
databank as EMBL. The trunk of the SVN tree at Berlios fixes all but one 
of them. I haven't had the time to fix this last problem yet, I hope to 
have the time this week.

To build the latest EMBL release you have to check out the source code 
from Berlios, build a new MRS and make sure you copy the stop.txt file 
from the mrs/plugin directory to the /usr/local/share/mrs/update_scripts 
directory. This file contains stop words to use and limits the size of 
the indices which is essential to avoid the last bug.
