I just did a quick test and found out that the query 'id:*_human' on  
swissprot takes half a second on my Mac Mini. I then tested with the  
soap_server I created for the SIMDAT project and that took only 5  
seconds. So I think the problem here is in the soap server script.

You could try to check out the trunk of the mrs project, there is a  
new soap server in there in the mrs/ws directory. It also has a WSDL  
that describes what you can do with it. If you think WSDL's are a bit  
hard to read in a text editor (I do) you can use e.g. Eclipse with  
Web-tools installed to get a graphical view or simply look at the  
mrsws.h file in mrs/ws/Sources/ to see what you can do.

I'm still developing this soap server so you can expect to see new  
and/or changed methods as time goes by.
