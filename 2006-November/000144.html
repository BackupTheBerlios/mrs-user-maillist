	Dear Maarten,

I hear that you are, amongst other things, developing a new and more 
efficient version of the Web services. I noticed that mrs-soap.cgi can 
perform a query and send back the ID's + optionally the description lines ;
it can also take a list of ID's and send back the complete entries. The 
experimental mrsws, developed for SIMDAT, does not have the option of 
sending the description lines. I find the possiblity to get the 
description very useful ; could it be recovered ? Also, for sequences the 
possibility to get the sequence in fastA format instead of the entry in 
"native" format would be useful. Of course, the possibility to get any 
data field would be even better, but I think that this would not be 
possible without a complete rewrite of the parser files.

Another idea : besides the WWW interface and the Web services interface, 
is it possible to write a Perl program that queries a local installation 
of MRS and gives the same kind of result as the Web services, but then 
faster, since it avoids the overhead of SOAP ?
