> first created under another name and only then all together "moved"

The new setup will use a dedicated daemon process that serves the  
entries using a SOAP protocol. I've designed the server so that it  
keeps the databanks loaded continuously. In that case it doesn't  
matter if you delete a file while it is being used since the file  
will actually be deleted only once the last process has closed it.  
The idea is to send a SIGHUP signal to the daemon once indexing is  
done and the daemon will reload the databanks.

But maybe it is better to simply copy over the data for UniProt,  
after all we're talking about ~5 Gb and that's peanuts. If I do it  
this way, there's no risk of having an unaccessible UniProt during an  
update.

As for EMBL, I think I will create this new update databank that uses  
the release databank and links to it. I will then also implement some  
fall back mechanism to use when the release databank has changed. I  
don't think that will be too hard to do.
