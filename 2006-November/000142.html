> date ; ./smrsget sprot 'id:*_human' > HUM ; date

To check if this is a performance problem in the query handling or a  
SOAP problem, you could repeat this with the following command to see  
if that times out. If it does, it is the perl script that causes the  
trouble since the results should be exactly the same.

date ; ./smrsget sprot 'ox:9606' > HUM ; date
