> The code is not completely stable yet.

I found out that an update I made to mrs since the release of version 4 
may cause crashes when doing a simple query. This only happens if you 
have a joined upgrade databank (like EMBL) and the update of the joined 
part of this databank went wrong.... Lots of if's there.

Anyway, if you experience random crashes right now (since the latest 
embl release update) and you have files in your mrs data directory whose 
name ends in a UUID, you should remove these files and rebuild them.
