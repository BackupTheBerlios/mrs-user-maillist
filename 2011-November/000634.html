Hi Georgios,

> > For the current version 5, is there an emboss method (emboss.default). 
> > I am not sure the mrs3 method is still there. Otherwise, what is the 
> > best recipe to achieve that?

I have checked that it is possible to access MRS servers using
individual DB definitions as it used to be before release 6.4.0. For
example, the following DB definition for embl database on MRS CMBI
server looks working.

  DB mrsembl [
    type:   "sequence, text"
    format: "embl, text"
    methodentry: "mrs"
    url: "http://mrs.cmbi.ru.nl/mrs-web/plain.do?db=embl&id=%
s&format=text"
  ]

Setting methodentry to 'url' should work as well.

You may try individual DB definitions until server support for MRS is
fully working in EMBOSS, which might be a matter of writing correct
server definitions and its cache/server files. I apologise if i'm
misreading your question.
