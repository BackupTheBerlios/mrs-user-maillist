Dear Guy,

> I just installed MRS version 5 under Wubi. When I try to run mrs-build I
> get the error message :
>
> /OPT/MRS/bin/mrs-build: error while loading shared libraries:
> libzeep.so.2.5: cannot open shared object file: No such file or directory
>
> The directory /usr/local/lib/ contains
> libzeep.a libzeep.so libzeep.so.2.5

I think this may be simple to solve. In Linux you have to tell the 
kernel where to find shared libraries, you do this with the ldconfig 
command. In the directory /etc/ld.so.conf.d you should create a new file 
containing the path /usr/local/lib. Then run ldconfig as root. After 
this, your mrs-build should work.
