Dear all,

I had some complaints about MRS recently. It appears that when you build 
MRS with gcc with 4.5 or 4.6, you'll end up with an executable that 
won't work properly. mrs-build fails on indexing the enzyme databank 
complaining about duplicate ID's.

Anyway, the only solution I see is to build MRS with gcc 4.4 for now. 
I'm not sure if I can motivate myself to locate the problem this time, 
it is not the first time I hit a compiler bug in gcc...

You can specify the compiler by editing the make.config file or as 
parameter to the configure script.
