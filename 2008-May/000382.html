	Dear Maarten, dear all,

This weekend I was testing something and got surprised by the fact that in the 
Pfam-A databank the field 'tp' contained a keyword 'tp'. I took a look and 
understood why. I take the occasion to send you a modified version of the file 
pfam.pm, with also and older fix for another problem :

- There are two lines with
    substr($line, 5)
    but the file goes
    #=GF TP   Domain
    so the text to be parsed starts at posion 11 rather then 6. Putting
    everywhere  $value  instead fixes it.
-  The data files for Pfam-B and Pfam-A-seed had been inverted

As well as with some enhancements, which you might or might not find useful :
- a list of long field names
- a numeric 'The number of sequences per cluster' field
- not indexing some fields with technical information about how the model was
   built
- a version section (to make it work the updater must also retrieve
   relnotes.txt)

	Regards,
	Guy Bottu,
	Belgian EMBnet Node
