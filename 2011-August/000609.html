Dear Uma Mahesh (మహేష్),

> We have a local installation of MRS. We are seeing patterns of high
> memory usage where MRS tomcat (java) is taking up >35G of total memory
> (both RAM and SWAP), and mrs-ws web service another 15G. I am using MRS
> for integrating ~4TB of data. The mrs index size is around a 1.5TB. Is
> it normal for MRS to consume such memory?

It is not uncommon. The mrs-ws application uses memory mapping for the 
blast indices and these can be large. The Java process uses quite a lot 
though, no idea why that is.

If you upgrade to mrs version 5 (I uploaded version 5.1 today to the PPA 
at https://launchpad.net/~hekkel/+archive/ppa/ ) you can dump the Java 
front end and use the new web interface. I believe it is better and faster.

Best regards,
