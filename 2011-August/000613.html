Op 17-8-2011 12:11, Georgios Magklaras schreef:
> I know this works for basic blast queries because when I submit one, I
> can see the blastall executable referenced properly in the defined path
> ('globin' as a search term, select a result from TrEMBL, click on Blast,
> verify the data entry is in Fasta format, submit BLAST query against
> 'PDB'). My understanding is that BLAST-ing against PDB uses the external
> blastall binary, whereas other queries against say 'UniProt' appear to
> show the mrs-ws handling things.

This may require some clarification. The choice to use NCBI blast is 
made based upon a parameter set in the constructor of the parsers. Look 
at the pdb.pm file and see at line 77 that the option 'blast' has as 
value 'ncbi'. When using 'ncbi' you also need to specify 'seqtype'.

Of course this means that MRS now is capable of blasting nucleotide 
databanks. As long as you edit your parser scripts to enable this. You 
have to add the options as described above, and then you of course need 
to add code to the parsers to call AddSequence with a DNA sequence. I 
won't do this at the CMBI since we don't have the hardware to allow DNA 
searches, but you're free to do so yourself of course.
