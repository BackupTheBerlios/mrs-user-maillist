Dear all,

I finally implemented an interface to the NCBI tools in MRS. This means 
it is now possible to use formatdb to create blast databanks and 
blastall to do the searching. Of course this also means that searching 
DNA is now possible too.

But now that this is possible, I have to update the user interface.

I see several options:

- Completely transparent dummy mode. This means I use heuristics to find 
out if the query was DNA or protein. Based on the outcome and the 
selected databank I choose a blastall program. This means that the 
background will use translation automatically.

- Limited dummy mode. Same as before, but translation is prohibited. 
This means the databank list will only show either DNA or protein 
databanks based on what the script thinks is entered in the query box. 
Reason to do this would be to avoid excessive blast load on the server.

- Manual mode. Make separate tabs for entering protein or DNA queries 
and separate tabs for DNA and protein databanks. This makes it 
immediately obvious to the user a translation is requested.

I think I prefer the first option myself. But I'd like to hear comments 
from others before I proceed.

-maarten
