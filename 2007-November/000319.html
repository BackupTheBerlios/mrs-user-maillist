> In the status log file there seems to be a clue:
> ... at /usr/local/share/mrs/parser_scripts//enzyme.pm line 100 - there
> seems to be a double forward slash.
> and most probably the error:
> Can't locate auto/MRS/MDatabank/StoreMetaDa.al in @INC ...
> 
> How can I best fix this problem?

Do you happen to have multiple versions of perl installed on your
computer? Shouldn't be a problem since the configure.pl script should
take care of that, but nevertheless.

And maybe you have an old MRS version located in some other Perl install
directory. If you have locate you might want to look for all occurrences
of MRS.pm and MRS.so on your computer and remove them before
reinstalling. If you don't have locate you might want to use find, like so:

find /usr/lib (or wherever your perl modules are) -name MRS.pm

If this doesn't help, please let me know.
