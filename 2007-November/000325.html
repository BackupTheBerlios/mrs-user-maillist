For those who are tracking the changes in MRS' trunk. Be careful. I just
updated the parserscripts, there are some changes you should review
before using them.

E.g. the PDB script now looks for .gz files instead of .Z files to
reflect the use of the new PDB server. The embl parser contains the
changes Guy Bottu made for separate update and creation date indices.
And finally, the WSFormatter.pm script was modified to spit out less
warnings.

The reason you need be careful is that if you have created an embl
release file with the old embl parser and create an update with the new
one, merging might fail due to differences in the indices. So you should
only start using the new embl parser when you're ready to do a rebuild
of embl release.
