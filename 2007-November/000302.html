> I have installed mrs-3.0.1b, i copy the stop.txt file from the mrs/plugin
> directory to the /usr/local/share/mrs/update_scripts directory. I did the make
> of the embl_releases without problems, it generate the file embl_releases.cmp
> but they do not appear in the web interface.
> The *.cmp files are ok, because i can run mrs.pl info -d embl_release.cmp
> without error. Any idea?

The files available in the web interface are determined by the
mrs_config.xml file. This file is usually located in /usr/local/etc.

I normally leave out embl_release. I use the embl_updates.make file to
create an embl_updates.cmp file and then merge the indices from
embl_release and embl_updates into embl.cmp. This last file then
contains the latest up-to-date data.

For this to work you will have to leave the embl_release and
embl_updates files around. And it breaks when the embl_release file is
updated... I'm going to redesign the way data is stored to avoid this
problem.
