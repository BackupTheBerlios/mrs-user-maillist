If you get back a message at all from the blast server that means the  
server is running. And yes I did implement my own blast algorithm as  
well as the queueing mechanism and all.

You could start the blast server but without forking it so it stays  
in the foreground. You can do so by starting it with the -f flags  
from within a terminal. And then you can specify the -v flag to get  
more verbose output. These two should give you some more feedback  
about what is going on.

There are some extra options that might influence the way blast  
works. E.g. there's a threads parameter in the mrs-config.xml file,  
you might want to change that to two e.g.
