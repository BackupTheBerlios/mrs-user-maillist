I work with rbahena at unam.

We managed to install mrs from svn (2007 nov 13) on a machine with amd 
processors and ubuntu.

We added a chomp($cpu) at the end of line 65 of configure.pl script.

Also, we used the --boost* options to configure.pl script like this:

$ sudo ./configure.pl --boost_regex=boost_regex 
--boost_filesystem=boost_filesystem --boost_thread=boost_thread

For what we saw, in ubuntu, the boost libraries (installed with apt-get) 
  *.so are symbolic links to *-gcc-mt.so

For the embl databases, after we installed the new version, we did what 
you told us to run the make for embl_updates, and it worked. So now we 
can see the Nucleotide (EMBL) in the web interface.


There is also an odd thing happening: if we search "human" in the EMBL 
database, and go to the first result, for example "bc016672", the 
Taxonomy links, for example "Eukaryota" (pointing to 
.../mrs-3/entry.do?db=taxonomy&query=Eukaryota) throw a "HTTP Status 
500" error. But if we search for "Eukaryota" on the Taxonomy database, 
it works ok.
