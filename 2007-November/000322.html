Maybe a similar problem, is the path to clustalw specified in the mrs- 
config.xml file? Is the user who runs the mrsws-clustal daemon  
allowed to launch clustalw?

What you could do is try to build a debug version of the clustal  
daemon. You can do this by running the command:

$ make DEBUG=1

in the mrs/ws/clustal directory. Then run this application with the - 
v flag and look at what is send to the terminal when you try to do a  
clustal from the web interface.
