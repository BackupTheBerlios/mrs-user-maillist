Guy,

Adding a case sensitive index is possible and not that much work.

The reason to allow multiple ID's by rewriting the duplicates is that  
it is very frustrating to have indexing fail for a databank that  
contains a duplicate ID. Esp. if it is e.g. Genbank and it took you  
three days to find out there was a duplicate ID. The current solution  
allows you to at least find the data even though the returned ID's  
don not make much sense.

Rewriting different case in ID's to something like LC_id can be done  
in the parser scripts. No change is needed in MRS itself to allow for  
this. Though I get the feeling that you trying to find a workaround  
for a problem that should be solved elsewhere.

-maarten
