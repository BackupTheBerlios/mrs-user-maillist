	Dear colleagues,

I just noticed the following. When I do a search at the BEN site 
(http://bendisk.ulb.ac.be/mrs) with as query
length > 10000
I get for the following databanks:
UniProt : no hits found
SwissProt : no hits found
TrEMBL : 63
At the "mother server" at RU the SwissProt and TrEMBL are not visible as 
such and I could only check that there too the query yields no entries 
for UniProt. I first thought there was a problem with numeric indexes, 
but I am now almost convinced the problem arises when in a composite 
databank the first member contains no entry that satisfies the query 
while the second does.

Did anyoene also notice this ?
