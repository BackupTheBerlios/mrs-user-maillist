> As far as GenBank is concerned : duplicate ID's are a chronic disease,

> MRS to call the primary key index by another name than 'id' ?

The 'id' name is reserved in MRS. It is used to find updated records  
in the update construct (embl_release|embl_updates).

However, there's nothing that prevents you to put the primary  
accession number in the id index of Genbank instead of the current ID
