	Dear friends,

At the Belgian EMBnet Node we have started to use MRS as sequence access 
method and with this a problem has become visible : the time-out of the 
Web Services. It is convenient to use the MRS Web Services to retrieve one 
or a few sequences and give them as input to a program. It does however 
not work if the sequences are with too many. E.g. a query id:*_mouse in 
UniProt is convenient to obtain all the mouse sequences, but in our hands 
it often times out.

A question for the developper : do you think the problem is inside MRS (and 
could be fixed) or is it due to the configuration of our httpd server ?

It would be nice if it worked, because MRS in its Web Services mode (I am 
not talking about the WWW interactive interface) could be a very useful 
and powerful tool.

	Guy Bottu,
	Belgian EMBnet Node
