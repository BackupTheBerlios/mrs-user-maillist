	Dear MRS users and support,

I humbly suggest some improvemnents for the PROSITE config files. You can 
download the modified versions from 
http://bendisk.ulb.ac.be/mrs/cgi-bin/mrs.cgi?dbinfo=prosite

The changes are :
- added "type" field (PATTERN, MATRIX or RULE). Certainly when the db 
  is used in a pedagogic context in can be useful to sort out the 
  different types of entries.
- prevented from indexing the MA, NR and PA fields, which contain no
  useful index words
- added a "version" routine
- added a hyperlink to PDB
- cleaned up the names of the fields in the info page

(Furthermore, I did not do it, but we might remove the to_fasta routine,
which it not used).

	Regards,
	Guy Bottu
