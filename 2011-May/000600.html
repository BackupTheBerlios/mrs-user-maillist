>
> * If I install it from the PPA, does it overwrite the current, 
> installed previous, version of MRS? I understand that if I run both I 
> would need to change the port numbers (that are, I believe, the same 
> in both versions) - but otherwise can the two versions co-exist? How 
> it is with previously indexed datasets; are they compatible or should 
> I re-index them for the new version?

The PPA won't overwrite the older version. That's because the PPA 
installs in /usr while the old version installed in /usr/local by 
default. Config file for the new version is in /etc/mrs and for the old 
one in /usr/local/etc. So theoretically you can share one data directory 
with two installations of MRS. And if you specify separate port numbers 
in the two config files you can even run the servers along each other.

You can also specify the config file to use from the command line for 
most of the tools.

The changes between 4 and 5 are basically summed up in a previous post 
to this list. But to summarize it:

- New installation method, and as a result new default installation 
directories. Also, the installation scripts assume there is a user 
called 'mrs-dba' who owns the created MRS files.
- The make script is now an executable i.e. you have to run 'mrs-update 
enzyme' instead of 'make -f /xxx/makefile enzyme'.
- New application server in mrs-ws, no longer need to run a Java web 
application.

And then there were quite a few minor bug fixes.

Hope this is sufficient to help you to start evaluating MRS 5? If not, 
please let me know. I don't think you need to change your MRS::Client 
code. If I changed the SOAP interface, I only exteded it. And if I broke 
it, I didn't do that intentionally. Honest.
