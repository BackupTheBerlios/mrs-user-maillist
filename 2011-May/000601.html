I have noticed that EBI's dbfetch server uses a similar approach as
well. dbfetch's REST response to metadata queries is very similar to
wsdbfetch's response to getDatabaseInfoList calls.

http://www.ebi.ac.uk/Tools/dbfetch/dbfetch/dbfetch.databases?style=xml

> However, if you're only interested in plain text records, you might as 
> well use the 'download' facility in MRS:
> 
> http://mrs.cmbi.ru.nl/mrs-5/download?db=sprot&id=104k_thepa
> 
> This returns the records in plain text. Maybe that's a better option for 
> you?

This might be a better option for plain text records, I will discuss it
with EMBOSS team. EMBOSS supports multiple data source formats when
available, it would be best if we use a facility that allows format
specification. I also have noticed that notion of format in mrs is a bit
different than it is in dbfetch. Actually dbfetch also have the concept
of style, for example 'html' is one of the styles in dbfetch while it is
one of the formats in mrs.
