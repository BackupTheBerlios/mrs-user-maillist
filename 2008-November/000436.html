Hi,

I finished a release candidate for version 4.1 today. You can download 
tar balls from ftp://ftp.cmbi.ru.nl/pub/software

This new version no longer needs gSoap to build, it uses libzeep from 
now on. I also changed the SOAP API, to make it easier to work with MRS 
from Python SOAP scripts. Other changes are the removal of libuuid.

I spent many days tracking down bugs that appeared to be coming from 
third party libraries and tools. Don't use e.g. gcc 4.3.2 since it will 
generate crashing code. And don't use the e2fstools libuuid, since that 
library creates a daemon process when called the first time. This daemon 
however does not close it's file descriptors and so blocks you from ever 
restarting the MRS webservices application.

The addition of libzeep makes it necessary to have at least boost 1.36 
installed which is not default on any recent OS AFAIK. You will thus 
have to build this library by hand. Download the source code and type 
the command:

bjam --toolset=gcc-4.2 link=static threading=multi release stage install
