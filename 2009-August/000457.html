	Dear all,

The wrappers4EMBOSS suite (http://wemboss.sourceforge.net) allows to 
integrate some 3rd party software under EMBOSS.

It contains a module MRS, which offers the following :
- a Perl script mrsindexsearch.pl, which allows to search in sequence 
databanks and retrieve the result as a List of Sequence Names, usable by 
EMBOSS, or as a databank in fastA format. mrsindexsearch.pl can be used 
interactively in a terminal. It actually has the look-and-feel of the 
program lookup from the now defunct GCG suite.
- an EMBOSS "wrapper" that allows to use the above mentioned 
mrsindexsearch under wEMBOSS or actually under any graphical user 
interface that can be used with EMBOSS. This makes it easy to use.
- a script that allows to use MRS as a sequence databank access 
mechanism for EMBOSS (so that you can e.g. type sw:*_human to let an 
EMBOSS program operate on all the human proteins from Swissprot).
- a Perl command line script mrsget.pl, which allows to perform a query 
in MRS query language on any kind of databank and retrieve the result in 
any kind of format (plain text, just the description lines, HTML as in 
the server,...).

The current version wrappers4EMBOSS 2.3.0 provides support for MRS 
version 3 as well as version 4 and it can work as well with a local 
installation of MRS (with an access to the Perl API) as with a remote 
MRS server (using a SOAP-Lite Perl Web Services client).
