A lot can go wrong.

First of all, make sure you run mrsws from the command-line. It  
should display a list of databanks it loads and then a line for each  
SOAP command processed.

The /usr/local/etc/mrsws-config.xml file contains the settings that  
define to what port mrsws listens, and what address. Perhaps you  
should change the address in this file for each server to '0.0.0.0',  
that way the server will listen to all addresses.

Then you will have to specify where the Java web application needs to  
connect to find the data. If tomcat runs on the same machine as  
mrsws, this should not be necessary. But if you do need to change it,  
use the Tomcat Admininstration tool. The variables you need to change  
can be found at:

Tomcat Server/Service (Catalina)/Host (localhost)/Context (mrs-3)/ 
Environment Entries/

Change the address and port number to the ones used in the /usr/local/ 
etc/mrsws-config.xml file.

Hope this helps?
