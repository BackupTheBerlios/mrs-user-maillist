Merging in stages is definitively supported. In fact, some users use  
this to merge over 600 MRS files into one.

You can use the -l option for 'mrs.pl create' in this case to avoid  
copying over all the data. This option then only creates the index  
and links to the data in the original files.

The option to search multiple databanks as if they are one (dba+dbb 
+dbc) is no longer supported since it was quite a performance  
penalty. And it is not possible to do correct ranked searches in such  
a concatenated set of databanks. On the other hand, search speed for  
individual databanks was considerably improved between 2 and 3 and  
merging doesn't take that long anymore either.

I do see the same problem as you mention here though and I might  
reintroduce the joined databank feature again. However, it is not a  
trivial thing to do and I will have to spend quite a bit of time to  
do it right.
