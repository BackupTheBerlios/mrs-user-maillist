> By the way, I just bumped on another problem : if you click on a "GO"
> link the entry is not retrieved, because an URL of type
> http://mrs.cmbi.ru.nl/mrs-3/query.do?db=go&id=7155
> is generated ; the script fails to padd the necessary O's so that  
> you get
> http://mrs.cmbi.ru.nl/mrs-3/query.do?db=go&id=0007155

This clearly is a bug in Xalan. Well, that's my opinion anyway.  
Here's the code in the style sheet:
<xsl:variable
	name="go"
	select="format-number(
			Gene-commentary_source/Other-source/Other-source_src/Dbtag/ 
Dbtag_tag/Object-id/Object-id_id,
			'0000000'
		)"
/>

As you can see I specify the format here and according to the XSLT  
documentation this should work.

In a future release of MRS I will integrate XSLT handling in the soap  
server so I can use a more up-to-date XSLT processor.
