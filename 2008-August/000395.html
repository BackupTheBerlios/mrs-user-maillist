I've finally squashed the indexing bugs that plagued the development 
version of MRS 4. I've created a first beta tag to celebrate this 
occasion. You can find the code at:

https://svn.cmbi.ru.nl/mrs/tags/V-4.0-b1/

Using a debug version I was able to index genbank release in some 20 
hours. No merging was needed in this process.

A problem might be that memory usage is quite high. At one of the latest 
stages the build process of genbank mrs-build used 22 GB of virtual 
memory and 17 GB of resident memory. Fortunately my server has 32 GB, I 
think you will need that amount too if you want to use the current 
version. Using swap space instead is not an option.

I will try to reduce the memory usage before the final release of MRS 4, 
but don't expect too much here. After all, it is this memory usage that 
makes MRS as fast as it is now.

-maarten
