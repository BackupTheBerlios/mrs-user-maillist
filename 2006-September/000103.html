Hello Guy,

> I noted that for EMBL and UniProt the fields are named de, os, kw, ft,
> etc., while for GenBank and RefSeq they are named definition,  
> organism,
> keywords, features, ... . Would it not be nicer to systematically  
> use the
> same field names for all databanks ? This would surely be helpful  
> for the
> development of scripts that use the MRS Web services. (I am actually
> writing such a script to search sequence databanks and generate List
> Files usable under EMBOSS).

The reason for the way MRS works is simple. It allowed me to write  
fault tolerant parsers. Whenever a new field shows up in a databank  
like SwissProt it is being indexed automatically into a new index. No  
user intervention needed here.

Of course, this can be extended so that we use a predefined list of  
index names along with a mapping to current fields in SwissProt and  
GenBank. But then someone has to come up with this mapping. I'd  
rather not make that list myself since I would be afraid to make  
mistakes.

There is another tiny problem associated with this, I always hear my  
colleagues talking about the 'DE' record, they are completely  
conditioned to think in two letter codes. If we change this they will  
have to learn to use the new names.

-maarten
