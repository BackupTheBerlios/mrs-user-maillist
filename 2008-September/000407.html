Since it looks like the indexing part of MRS 4 has now been finished, I 
will now concentrate on the other parts that need polishing.

One question that pops up immediately is how many of you are using the 
Perl interface? And how?

The MRS 3 interface was very important, it was for many the only way to 
use MRS. Now, with MRS 4, you use a separate application to build 
databanks and this app will load the Perl based parser scripts.

Is anyone using the Perl interface to create databanks? Do you want this 
feature in MRS 4? If I don't get a positive answer on this, I will 
remove this functionality. The Perl interface will then remain for doing 
queries and blast searches from within your Perl scripts.

As an alternative I can write an MRS 3 API compatible perl module that 
can be used instead of the MRS 3 one but which uses SOAP to do the 
actual searching.

I would really appreciate any comments on this.
