Normally (I assume you are still under 3.x, if you are at 4.x I don't 
know, haven't tried that), when something bad happens you get lots of 
indermediate files that the system creates for sorting, etc like:

databasename-[number].cmp.XXlDjejb.

The best way is to clear these files and re-index but in your case, if 
you are memory starved, I am not sure this is a good idea. You are not 
the only one facing this problem, so I 'll tell you what: I shall place 
a copy of the indexed embl release 95 with updates here:

ftp://ftp.no.embnet.org/biodatabases/mrsindex/

Feel free to hit the server, we don't mind. But wait for the next hour 
or so, until all files sync with the FTP server (the copy is in 
progress). They need to be placed in the mrsdata mrs dir.

I hope that this helps, and if more people face these problems, it might 
be an idea to maintain this dir in the ftp server, as not all have 
systems packed with memory and I read from Marteen that version 4 is 
going to be even more memory hungry.
