George,

I've created a CentOS version of MRS and I'm about to release this as a RPM. 
I've used the 5.3 version of CentOS and had to install several libraries by 
hand since they were either missing or too old. Therefore I had to link the 
executables statically against these libraries.

This raises a new point though, I cannot create a binary perl plugin since 
my hand-build versions of the libraries are not position independent (-fPIC) 
but the libperl.a is and mixing those is not allowed. Since using the perl 
plugin is a deprecated technology in my eyes anyway, would it be a real 
problem leaving this one out of the RPM?

Another potential problem is libstdc++, since I've used CentOS 5.3 I hope I 
can assume that everyone has the right version installed? Trying to link 
that one statically is proving to be difficult.

-maarten
