Hi,

I wonder what is the best way to iterate over all entries in a particular
databank (when doing some data mining).

Ideally, I would like to do it via the SOAP interface - the same way as the
mrs-web application does it - but unfortunately, the SOAP interface does not
keep any state - so every iteration seems to make a new search (at least I
think so). Which takes time. Are there any plans to add this iterative
approach to SOAP perhaps?

What work fine and fast, is the MRS.pm plugin. This program reads the whole
sprot (514212 documents) in about 30 seconds on my laptop:

use MRS;
my $db = new MRS::MDatabank ('sprot')
    or die "Could not open databank sprot\n";
if (my $r = $db->Find ("*")) {
    while (my $id = $r->Next) {
        my $doc = $db->Get ($id);
    }
}

But the MRS.pm plugin can be used only on the same machine where the data
are (am I right?).

Thanks for your comments and suggestions about this issue.
