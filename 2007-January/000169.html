Hi,
Yesterday, together with David, we tackled the problem of installing a 
valid gsoap package on our SuSE, Fedora Core and Gentoo boxes. As I 
reluctantly  suggested the day before, we tried to install the same 
gsoap release as is available as Debian package, namely 2.7.6d (we 
guessed that the Debian people also encountered problems with more 
recent gsoap packages, which might explain why they are still at 2.7.6d, 
now more than a year old...).
And indeed this version, downloaded as  platform-independent  package, 
compiled properly on a FC5 32 bit and a Gentoo 64bit machine. Alas, the 
make install did not preperly complete its work. Functionality required 
some symlinking. I then tried 2.7.6e on FC5 and this one succceeded 
completely : ./configure, make and make install completed without error 
and allowed me to completely compile, link and install MSR3. Same was 
the case on the Gentoo 64 bit.
I hope this info was helpful.
Did anybody manage the gsoap problem in a different way ? Details would 
be helpful.
Robert
