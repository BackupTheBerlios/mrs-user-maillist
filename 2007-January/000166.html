The webserver application is a daemon. If you update your svn code  
again you have the -b option that forks mrsws and puts it into the  
background.

About mrsws:

- There is a mrs-config.xml file that contains the config parameters.  
The default location is /usr/local/etc but this can be changed in  
several ways.
- when running in daemon mode, mrsws writes to a log file. This file  
is located by default in /var/log/mrsws.log.
- sending a SIGINT (killall mrsws) will tell mrsws to quit gracefully.
- sending a SIGHUP (killall -HUP mrsws) tells mrsws to reload all  
databanks.
- if you have specified mrs-config.xml file, or this file exists in / 
usr/local/etc then the databanks listed in this file will be loaded.  
Otherwise all mrs files in the data directory are loaded.
- if you specify an address 'localhost', mrsws will bind to 127.0.0.1  
and is not available from outside the machine.

The java interface is not available yet. I will release an alpha for  
that sometime this week, I hope.
