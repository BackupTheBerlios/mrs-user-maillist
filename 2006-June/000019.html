> Hi,

> Should I automate the setup of the website too? And how far should

>
> - I can replace the shebang at the top of the cgi scripts with the

Yes. This helps

> - I can copy the web site to some location defined by the user

I like the MrsWeb path. It keeps good humour around MRS installers ;-). 
So you suggest this as default and allow for change if installer insists 
on it.

> - I can create a mrs.conf file for the user

Which mrs.conf ? That one from the cgi-bin with the list of installed 
databanks ? No leave this for another round. The managers should know 
what to declare and each paragraph is rather self-explanatory.

> - I can hack the htttpd.conf file to add mrs

This I would not do. We have apache 2.2.2 on our boxes (not from vanilla 
distibutions) and the script and scriptalias lines you mention are not 
sufficient. You require in addition the declaration of a  
<Directory>...</Directory> structure to get it working. Also recent 
Apache installation recommend NOT to touch httpd.conf, but use includes 
from separate config files.

> - I can find or copy an installation of jmol for use in the structure  
> pages

This is a structure visualiser, isn't it. Nice to have but should be 
optional.

> - I can try to find out whether the user has clustalw installed and  
> enable/disable the page accordingly?

Nice to have. Same as preceding.

> - I can try to find out whether a queueing system is available and  
> use that for blast...
>
Certainly useful, especially when blastn and blastx, tblatsn and friends 
will be there. But this can be left out for "advanced" installations.

> As you can see this is on an increasing scale of complexity. Maybe  
> good documentation is of more help here than a script that tries to  
> do everything but that will probably fail due to unforeseen problems.
>
A very interesting option, as long as you don't have a home brew blastn 
(and  friends...), would be to allow for calls, from a MRS query result 
list, of an independent NCBI Blast. I guess this has been the case on 
some cmbi installation. I am wrong ?
I agree that the users should not be encouraged to run blastn as their 
first option (most badly trained tend to do so, and end up with heaps of 
useless info...)
