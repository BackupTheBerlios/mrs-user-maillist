>
> On the vanilla SuSE 10.0 and Fedora Core 5 installations, gzcat is  
> unknown. This is the reason why e.g. "make uniprot" chokes on a  
> fresh MRS_2.0 installation. Putting a logical link to zcat solves

Duh...

I knew this would happen, since I burned myself on this topic several  
times. Problem is that as you may remember there used to be two tools  
for compression on UNIX systems, compress (with zcat) and gzip (with  
gzcat). However, since gzip is now capable of reading compress files  
the zcat tools seems to be dropped and gzcat is renamed to zcat.

In make_xxx.conf you can find a statement:

GZCAT = /bin/zcat

Unfortunately I didn't use the variable $(GZCAT) everywhere, the file  
make.post contains several statements that still use gzcat. I've  
fixed that, will check this in later today.
