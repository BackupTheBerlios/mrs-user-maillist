Hi Maarten and friends,

I want to mention two issues that can easily be cleared about MRS_2.0 :
1. the failed_dbs.txt file in the status directory has to be 
world-writeable for apache to be able to send the edition command to 
clear a "failed" databank in the databanks.cgi view
2. I think that the pdb.make is not really as functional as most other 
make files. Starting the rsync by hand succeeds if the target directory 
exists. I am presently rsync-ing the PDB. I shall see tomorrow if the 
updating process works.

I have activated the lines in uniprot, embl_release and embl_updates 
make files in order for the software to build the NCBI-formatted blast 
files. The building works flawlessly once the formatdb command from the 
BLAST package is in the path. Also, I had to hand-compile 
swiss2fasta.cpp in the update_scripts directory and put it in the path. 
In this way, an external BLAST server can be fed permanently with up to 
date blast-formatted databanks.

Maarten, you mention in the end of the compilation phase that to keep 
databanks up to date, one should run a make daily and a make weekly. 
These scripts, albeit simple to compose, might be useful to be part of 
the distribution for people to use and enjoy.
