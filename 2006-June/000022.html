Op 27-jun-2006, om 7:50 heeft Robert HERZOG het volgende geschreven:
>
> Now that MRS_2.0 is  nicely indexing embl_release in less than 30  
> hours,and while Maarten is finishing up the website part of the  
> soft, it is time to start thinking about how to let partners take  
> profit of the modularity of MRS. MRS can be configured to build any  
> databank in a number of sections (32 at this time for embl, in  
> order not to overload the memory). By default, these sections are  
> wiped away after merging on the primary building machine. But this  
> can be prevented (just comment out the "rm" line in the  
> corresponding script !). Having these sections on a peer-to-peer  
> filesystem might be a great exercice in sharing. While Uni.  
> Karlruhe is building a _secured_ P2P filesystem, public solutions  
> like BitTorrent is a tempting approach suggested by David. What do  
> you think, David and George ? And Lubos, are you on the list ?

I've never used BitTorrent so I don't know how this would work out.

But I can give some remarks.

First of all I will have to implement a better balancing algorithm  
since the parts in EMBL differ extremely in size. Genbank is nicely  
balanced and EMBL should be the same to make it useful to divide the  
indexing work among other computers.

BitTorrent might be nice to have. But the question is, how many sites  
will have MRS files available? And will that be enough to make the  
use of BitTorrent worthwhile? Besides, it would be nicer if after the  
release of a large databank like EMBL all sites that volunteer to  
share in indexing agree on what part they will index to avoid doing  
duplicate work. That also means there needs to be one server that  
divides the tasks to the indexing sites and collects the location and  
ID of the parts that are finished so that others know where to pick  
them up.

A fair bit of programming and testing will be required to do  
something like that.

And then the question is, will it pay back? I mean, downloading the  
raw files takes many hours. Downloading the MRS parts from other  
sites will take even longer. Indexing the files yourself might be  
faster, especially if you don't have a fast link.
