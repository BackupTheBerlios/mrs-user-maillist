>
> pursuing my installation experiments, it seems that a GCC 3.3.4  
> compiler is not OK for building MRS_2.0.

3.3 is a bit old, I don't remember when I last used it. I do believe  
that 3.4 should work though.

> I'll consider installing GCC4.x on that machine, as recommended.  
> But this is a "production" machine and thus, changing the C  
> compiler is bound to produce... surprises.

The differences between 3 and 4 are not as big as between 2 and 3.  
The main thing that changed is standards compliance. However, maybe  
the ABI has changed and if so you might run into problems simply  
replacing your current compiler if you use plugins written in C++.

But there is a simple solution, simply download the source code for  
gcc, use configure to specify a command suffix like '-41' and you end  
up with a 'c++-41' named compiler that can be used alongside your  
current compiler. That's the way I do it myself and it works very well.
