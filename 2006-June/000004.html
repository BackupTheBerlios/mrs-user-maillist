I'm now working on the installation part of the db-update scripts.  
Doing that I realise I'm not so sure about the default location for  
all my stuff. There are several options, one is to have all MRS  
related stuff in a separate directory tree, like this:

/prefix/mrs/
/prefix/mrs/bin/mrs.pl
/prefix/mrs/bin/mrs_blast
..
/prefix/mrs/data/
/prefix/mrs/data/raw/
/prefix/mrs/data/mrs/
..
/prefix/mrs/parser_scripts/
/prefix/mrs/update_scripts/

That's a bit like SRS. I don't like it myself, but it makes  
installation a bit easier. However, in this case you need to have a  
settings file that needs to be 'sourced' at startup to be able to  
find all the apps and data.

An alternative approach would be to store the binaries in the default  
OS location for third party software. That's /usr/local/bin for most  
of them. In that case we could create a /usr/local/etc/mrs.conf file  
that contains the global settings for mrs on your system. This will  
also remove the need for using environmental variables. The /usr/ 
local/etc/mrs.conf file would then contain all the paths to other  
tools and data directories. And all the tools would use this  
configuration file.

As an extra I could build code that looks for an alternative .mrsrc  
configuration for user overrides of the settings.

What's your opinion?
