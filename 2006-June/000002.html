Maarten L. Hekkelman wrote:

> Hi,
>
> Let's start using this mailinglist for future communication regarding  
> MRS issues. That way we create a publicly available archive of  
> information for future users.
>
> I've updated the 2.0 release branch in svn and checked in a new  
> install script. It is not finished but I would like to get some  
> feedback on what it does. Right now it tries to find out what you  
> have installed on your computer and writes a make.config file that  
> should allow you to build and install the plugin and three  
> executables: mrs.pl, mrs_blast and mrs_query.
>
> The procedure to get this code is as follows:
>
> First get the source code from the repository at berlios:
>
> svn co https://svn.berlios.de/svnroot/repos/mrs/branches/2.0 mrs_2.0
>
> That will create a folder containing the release 2.0 code for MRS.  
> Now to test the installation you should copy out the code to a new  
> location with the following command:
>
> svn export mrs_2.0 mrs_test_2.0
>
> The new mrs_test_2.0 directory is what will finally be delivered as a  
> download. In this mrs_test_2.0 directory you can then run
>
> perl install.pl
>
> and follow the directions on screen.

Hi Maarten,

The installation went flawlessly on a "naive" freshly installed SuSE 
10.0 computer on Pentium architecture. This machine has
Linux kernel 2.3.13-15-smp
Perl 5.8.7
gcc 4.0.2
no swig
boost 1.33.0
boost-jam 3.1.4
all from distri, updated to latest fixes.

After starting perl install.pl, I got a few complaints about lacking 
Perl modules (URI, SOAP::Lite, etc.). I could cure all these by 
installing additional modeuls from the SuSE distribution. Only 
XML::SAX::Expat required access to the CPAN server (CPAN  >> install 
XML::SAX::Expat).
After that, a cd to plugin, the make and make install went with only two 
minor complaints about
/bin/sh: - : invalid option
but this did not choke the compilation.

So far so good !
